# Bundle an agentskills.io-compliant skill as a public CLI artifact

The Frame CLI ships a `skills/frame-cli/SKILL.md` file inside the published npm tarball, intended for AI coding agents that drive the CLI on a user's behalf. The file follows the [agentskills.io](https://agentskills.io) specification (frontmatter with `name` / `description` / `compatibility` / `allowed-tools`, body under 500 lines, single source of truth for the canonical event list and the deprecated→canonical redirect map). Agents install it via `npx skills add Frame-Payments/frame-cli` (skills.sh resolves the GitHub path) or by reading the file directly out of `node_modules/@framepayments/cli/skills/frame-cli/SKILL.md` after a normal install. The skill is a public contract: its `description` field, listed commands, and documented workflows are part of the CLI's public surface in the same sense that `frame --help` is.

## Considered options

- **No bundled skill; agents read `frame --help` and the README** — rejected. `--help` output is terse by design and per-command, not a coherent procedural manual. Agents that haven't seen the CLI before reinvent the same wrong patterns each run (awaiting `frame listen` as a foreground process, trying `frame customers create`, missing the `--json` parsing path on `logs tail`). The "Gotchas" content from agentskills.io best-practices has nowhere to live.
- **Bundle the skill as a URL only, hosted on docs.frame.com** — rejected. Breaks offline use, ties the skill's update cadence to a separate publishing pipeline, and means the version of the skill an agent loads can drift from the version of the CLI it's driving. Bundling in the npm tarball pins them together by construction.
- **Bundle a skill but invent our own format** — rejected. agentskills.io is the de-facto standard (Anthropic origin, adopted by Claude Code, Cursor, pi, and others; skills.sh as the install registry). Deviating buys nothing and forfeits skill-discovery from every compatible client.
- **Bundle an agentskills.io-compliant skill at `skills/frame-cli/SKILL.md`, distributed in the npm tarball, installable via skills.sh** — chosen. Matches the `linear-cli` pattern (which is the closest precedent in our peer set), pins the skill's version to the CLI's version, and surfaces the skill through the existing skills.sh discovery channel without us running infrastructure.

## Consequences

- The SKILL.md `description` field, the listed commands, and the documented workflows are public contract. Changing the description is a versioned change, not a docs tweak — agent harnesses index on description for triggering, and breaking the trigger silently regresses every downstream agent. Treat it the same as a CLI flag rename.
- `package.json#files` must include `skills/` (and `fixtures/`, since the bundled fixtures are referenced by the workflows the skill teaches). A regression test (`test/skill.test.ts`) parses the frontmatter on every `npm test` to catch frontmatter breakage and the 1024-character description limit.
- The 16 canonical event codes and the deprecated→canonical map appear in three surfaces: `src/commands/trigger.ts` (runtime), `frame trigger --help` (injected from the same arrays via `addHelpText`), and `skills/frame-cli/SKILL.md` (hand-written but covered by the fixtures-vs-`SUPPORTED_EVENTS` parity test). The arrays in `trigger.ts` are the single source of truth; the SKILL.md drifts only if a contributor edits the file without updating the test, which the test catches.
- The skill documents *only what ships*. Aspirational v1 commands from `docs/plans/cli-v1-client.md` do not appear in the SKILL.md until they land — an agent that loads the skill and then fails to invoke a documented command is worse than no skill.
- Adding a new command becomes a three-surface change: implementation, `--help` example, SKILL.md entry. The PR template enumerates the three places to update; a shared examples module was rejected as over-engineering at this surface size.
- Removing or renaming the skill, or moving it out of the tarball, is a breaking change for any agent that has cached `npx skills add Frame-Payments/frame-cli` resolution. If the package is ever re-scoped or the GitHub repo moves, a redirect ADR will be needed.
